🎯 نمای کلی سیستم

سیستم احراز هویت این پروژه شامل چهار بخش اصلی است:

1
ثبت‌نام کاربران جدید
دریافت اطلاعات کاربر، اعتبارسنجی و ذخیره‌سازی امن
2
ورود کاربران موجود
بررسی اطلاعات ورود و تنظیم وضعیت احراز هویت
3
محافظت از صفحات
استفاده از middleware برای کنترل دسترسی
4
مدیریت وضعیت
نگهداری اطلاعات کاربر در سراسر اپلیکیشن
✅ ویژگی‌های امنیتی: رمزگذاری کوکی‌ها اعتبارسنجی داده‌ها محافظت CSRF هدرهای امنیتی

🏗️ انواع داده‌ها و Interface ها

User Interface
interface User { id: number email: string username: string name: { firstname: string lastname: string } phone: string address: { street: string city: string } }
RegisterUserData Interface
interface RegisterUserData { email: string username: string password: string firstname: string lastname: string phone: string street: string city: string }
AuthContextType Interface
interface AuthContextType { user: User | null login: (username: string, email: string, password: string) => Promise<boolean> register: (userData: RegisterUserData) => Promise<boolean> logout: () => void updateUser: (user: User) => void isLoading: boolean }
⚠️ نکته مهم: تمام interface ها در فایل shared/types/auth.ts تعریف شده‌اند و در سراسر پروژه قابل استفاده هستند.

📝 فرآیند ثبت‌نام

مراحل ثبت‌نام:

1
دریافت اطلاعات کاربر
نام، ایمیل، رمز عبور، شماره تلفن و آدرس
2
اعتبارسنجی داده‌ها
بررسی فرمت ایمیل، قدرت رمز عبور و سایر فیلدها
3
ذخیره‌سازی اطلاعات
ذخیره در localStorage و تنظیم کوکی امن
4
هدایت به حساب کاربری
انتقال خودکار به صفحه محافظت شده

قوانین اعتبارسنجی:

// اعتبارسنجی رمز عبور const validatePassword = (password: string) => { if (password.length < 8) return "حداقل 8 کاراکتر" if (!/(?=.*[a-z])/.test(password)) return "حداقل یک حرف کوچک" if (!/(?=.*[A-Z])/.test(password)) return "حداقل یک حرف بزرگ" if (!/(?=.*\d)/.test(password)) return "حداقل یک عدد" return undefined } // اعتبارسنجی ایمیل const validateEmail = (email: string) => { const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/ return emailRegex.test(email) ? undefined : "فرمت ایمیل نامعتبر" }

🔐 فرآیند ورود

مراحل ورود:

1
دریافت اطلاعات ورود
نام کاربری، ایمیل و رمز عبور
2
بررسی اطلاعات محلی
مقایسه با اطلاعات ذخیره شده در localStorage
3
تلاش API (در صورت نیاز)
استفاده از FakeStore API به عنوان fallback
4
تنظیم وضعیت احراز هویت
ذخیره توکن و به‌روزرسانی state

منطق ورود:

const login = async (username: string, email: string, password: string) => { // بررسی اطلاعات محلی const storedUsername = localStorage.getItem('registeredUsername') const storedEmail = localStorage.getItem('registeredEmail') const storedPassword = localStorage.getItem('registeredPassword') if (username === storedUsername && email === storedEmail && password === storedPassword) { // ورود موفق با اطلاعات محلی setAuthToken() return true } // تلاش ورود از طریق API const response = await fetch('https://fakestoreapi.com/auth/login', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ username, password }) }) return response.ok }

🛡️ Middleware و محافظت از صفحات

عملکرد Middleware:

1
بررسی توکن احراز هویت
جستجو در کوکی‌ها و هدرهای درخواست
2
محافظت از صفحات خصوصی
هدایت کاربران غیرمجاز به صفحه ورود
3
هدایت کاربران وارد شده
جلوگیری از دسترسی به صفحات ورود/ثبت‌نام
4
تنظیم هدرهای امنیتی
اعمال محافظت‌های اضافی امنیتی

کد Middleware:

export function middleware(request: NextRequest) { const { pathname } = request.nextUrl // بررسی وجود توکن const authToken = request.cookies.get('auth_token')?.value || request.headers.get('authorization')?.replace('Bearer ', '') const isAuthenticated = !!authToken // محافظت از صفحات خصوصی if (pathname.startsWith('/account')) { if (!isAuthenticated) { return NextResponse.redirect(new URL('/login', request.url)) } } // هدایت کاربران وارد شده if ((pathname === '/login' || pathname === '/register') && isAuthenticated) { return NextResponse.redirect(new URL('/account', request.url)) } // تنظیم هدرهای امنیتی const response = NextResponse.next() response.headers.set('X-Frame-Options', 'DENY') response.headers.set('X-Content-Type-Options', 'nosniff') return response }
✅ مسیرهای محافظت شده:
  • /account/* - صفحات حساب کاربری
  • هدایت خودکار از /login و /register برای کاربران وارد شده
